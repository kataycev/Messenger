# Мессенджер

Данный проект представляет собой мессенджер, в котором реализованы следующие страницы:

- Страница регистрации и авторизации
- Пользовательская страница
- Страница админа

## Функционал

### 1. Страница регистрации и авторизации
Эти страницы предназначены для регистрации нового пользователя и входа в мессенджер. Они включают в себя:
- Поля ввода: **username** и **password**
- Кнопки:
  - **Войти**
  - **Зарегистрироваться**

### 2. Пользовательская страница
Это основная страница для пользователей, на которой они могут:
- Просматривать сообщения, присланные другими пользователями
- Искать пользователей
- Открывать и отправлять сообщения

### 3. Страница админа
Страница для администраторов, доступ к которой возможен только по специальному **username** и **password**. В этой панели админ может:
- Просматривать список зарегистрированных пользователей
- Просматривать отправленные сообщения между пользователями
- Отслеживать сохраненные действия пользователей

## Архитектура

Проект состоит из двух основных частей:

### 1. Front-End часть
Front-End — это визуальная часть мессенджера, с которой взаимодействует пользователь. Вся пользовательская логика и интерфейсы, такие как страницы регистрации, авторизации, обмен сообщениями и т. д., написаны на **JavaScript**.

### 2. Back-End часть
Back-End — это серверная часть проекта, отвечающая за обработку данных и взаимодействие с базой данных. Взаимодействие между Front-End и Back-End осуществляется через **HTTP запросы** и **веб-сокеты**. Вся серверная логика реализована с использованием **Python**.

## Технологический стек

- **JavaScript** — для Front-End части
- **Python** — для Back-End части
